Configuration

Tout d'abord, il faut configurer votre application. Pour cela, rendez-vous dans

FLFramework/Config/bootstrap.config.php

Ici vous pouvez configurer une multitude de chose.

  • cores\Sql:: La connexion à la base de données
  • cores\Router:: Les options par défauts du router
  • cores\Cache:: Les options d'admin du cache
  • cores\Log:: Les options d'admin des logs
  • define('__site_url__', "URL"); L'url spécifique du site si nécessaire
  • App:: Ces fonctions vous permettent de choisir si vous souhaitez utiliser une base de données ou non, le module d'url rewriting ou non...etc.

Dans le fichier :

FLFramework/Config/routes.config.php

Se trouvent la définition de tous les path indispensables pour le développement afin de vous faciliter la vie

Modules natifs

Quelques modules sont intégrés nativement dans le framework et permettent certaines choses importantes

FLFramework/Module/DALGenerator

Le DALGenerator va vour permettre de générer vos classes de manipulation de la base de données. Toutes les informations techniques nécessaire sont fournies à cette url :

http://YOUR_URL/Module/DALGenerator/index.html

Cet outil vous permet une totale libertée dans la création de votre base données et dans vos pattern de nommage. En effet, l'application ira chercher toutes vos tables de votre base de données pour en créer des classes miroir implémentant toutes les fonctions de base nécessaire comme :

  • getAll()
  • getBy($key, $value)
  • add($object)
  • update($object)
  • delete($id)

De même, ce programme va générer les modèles images de vos tables appelées "Entity"

Il est préférable de stocker ces classes ici :

FLFramework/Data

Puisqu'y résides déjà les trois classes indispensables au routing

Dans ce même dossier réside un dossier Base. Ce dossier contient les classes parents de ces modèles et contiennent toutes les fonctions citées précédemment. Ils ne doivent pas être modifiés puisqu'ils seront regénérés à chaque lancement du DALGenerator. Si vous souhaitez ajouter des fonctions ou des attributs. Il suffit de modifier les classes filles présentes dans :

FLFramework/Data/Entity

Et

FLFramework/Data/Repository

Les fichiers présents dans ces répertoires ne seront jamais ecrasés.

 

 

 

 

FLFramework/Module/NativeSQLModule

Ce module est quasi indispensable au bon fonctionnement du framework. Il ajoute à celui-ci les fonctionnalités de réécriture d'url et de multilinguisme. Il vous sera d'ailleurs rappelé (en mode debug) d'importer ces scripts SQL pour votre bien !

Ce module consiste en trois fichiers SQL (un par rôle) à importer dans votre base de données.

Les exceptions natives

Les exceptions natives sont dans le dossiers ci-dessous :

FLFramework/Library/Adapter

Il en existe quatre :

  • connexionException
  • controllerException
  • viewException
  • emailException

Elle gère les exceptions basiques du site. (connexion à la BDD impossible, controller introuvable, layout introuvable...etc)

Ces exceptions sont gérées par la classe error.php située dans :

FLFramework/Library/Core

Elles sont désactivables en mettant le mode debug à false ou en réglant les problèmes soulevés.

Les helpers

Les helpers sont des classes encapsulant certaines fonctionnalités utiles à un site web et disponibles dans les controllers ainsi que dans les vues.

Voici les helpers existant

  • FLFramework/Library/Helper/auth.php Ce helper gère la couche authentification de l'utilisateur (connexion/deconnexion/récupération...etc.)
  • FLFramework/Library/Helper/cdiscount.php Ce helper permet de requêter l'api cdiscount
  • FLFramework/Library/Helper/form.php Ce helper gère la couche formulaire (génération automatique, réception d'information...etc.)
  • FLFramework/Library/Helper/email.php Ce helper gère la couche email (création d'un e-mail, envoie, template...etc.)
  • FLFramework/Library/Helper/paginator.php ce helper permet d'implémenter facilement des paginations dans vos pages
  • FLFramework/Library/Helper/paypal.php ce helper permet d'implémenter facilement du paiement paypal dans vos pages
  • FLFramework/Library/Helper/rss.php Ce helper permet de générer des flux rss
  • FLFramework/Library/Helper/session.php ce helper gère la couche session
  • FLFramework/Library/Helper/token.php ce helper permet de gérer les token dans votre application (génération/comparation...etc.)
  • FLFramework/Library/Helper/upload.php ce helper permet de gérer l'upload de fichier

Toutes ces classes sont commentées et très facile d'utilisation.

De plus, il existe un controller TestController implémentant une action par helper afin de montrer l'utilisation de ces classes

FLFramework/Website/Controller/TestController.php

L'application

Normalement, vous n'aurez à toucher que le dossier :

FLFramework/Website

Ce dossier contient votre application.

  • FLFramework/Website/Content Contient vos assets (javascript, css, media...etc.)
  • FLFramework/Website/MVC Contient votre site web découpé en MVC (Model, Vue, Controleur)
  • FLFramework/Website/Partial Contient les vues partielles

Les plus frontend

FLFramework implémente nativement Sass (compass), afin de faciliter l'intégration. La logique est la suivante. Le dossier :

FLFramework/Website/Content/Sass/Base

Contient les Css de base tels que la feuille de reset ou encore les classes récurentes. Ensuite dans le dossier :

FLFramework/Website/Content/Sass

Les fichiers préfixés d'un '_' sont des fichiers voués à être inclus dans un fichier principal. Enfin, les fichiers non-préfixé sont les fichiers dit "d'index", incluant tous les autres dans un ordre défini


Le framework implémente nativement grunt. Il vous suffit de lancer les commandes si après pour en bénéficier (sous condition d'avoir déjà grunt et npm dans votre path)

npm install

grunt

Le gruntfile présent dans l'application permet de compiler tous les JS ensemble et de les minifier et de compiler tous les SCSS ensemble et de les minifier puis de les transformer en fichier css dans le dossier :

FLFramework/Website/Content/Css

Il implémente aussi jshint qui compile vos JS, ainsi que watch, qui vous permet de lancer toutes ces tâches en temps réel, dès qu'un fichier est modifié.

Enfin, il implémente le module img qui permet de compresser toutes les images de votre projet.

Petite précision, les JS et CSS présents dans le dossier :

FLFramework/Website/Content/Css/Module

FLFramework/Website/Content/Js/Module

Ne seront pas compilés automatiquement puisque c'est vous qui déciderez d'inclure tel ou tel module.

En parlant de ces dossiers module, ce sont des librairies JS très utiles. Une demonstration de ces plugins est présent dans :

FLFramework/Website/View/Test/jslibrairies.php

Le framework implémente jQuery, jQuery UI et Bootstrap. Libre à vous des les inclures ou non.

Pour info, ces librairies de base sont disponible ici :

FLFramework/Website/Content/Js/Base

FLFramework/Website/Content/Css/Base

Plus sur le Website

Dans chaque vue, vous pouvez inclure des choses différentes à deux endroits. Dans le <head> et à la fin du DOM dans le footer du site grâce aux sections.

$this->BeginSection();

Cette ligne signal le début d'une section.

$this->EndSection('sectionName');

Cette ligne signal la fin d'une section en précisant son nom.

Pour rendre le rendu d'une section quelque part dans le layout, il suffit de faire :

$this->Render($this->Sections['sectionName']);

Il existe plusieurs type de layout nativement. Le premier (default) est le layout par défaut et le principal.

Il existe aussi un layout pour les emails, un pour le rendu XML (rss) et l'autre pour le rendu json.

 

Il existe aussi un controller pouvant servir de Webservice servant du JSON. Ce webservice est ici

FLFramework/Website/Controller/WebserviceController.php

Et fait appel au layout json justement.

 

Ce framework permet aussi le tracking via Universal Google Analytics.

Le fichier de tracking se situe ici :

FLFramework/Website/Partial/tracking.php

Il suffit simplement d'y mettre votre UA-ID
